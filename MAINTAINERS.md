# Maintainers

## Project Status

HookMon was created by Dave Lenrow (Illumio) and released as a reference implementation. It is **not actively maintained**. There is no roadmap, no issue SLA, and no commitment to review pull requests or respond to bug reports.

This is intentional, not neglect. The purpose of this release is to demonstrate that the monitoring gap described in the accompanying blog post is real, that the detection approach is sound, and that the implementation barrier is low. The code is the proof. What happens next is up to the community.

## If You Want to Use It

Clone it, deploy it, use it. The Apache 2.0 license places no restrictions on use.

## If You Want to Modify It

Fork it. That's the intended path for anyone who wants to extend the sensor coverage, add SIEM integrations, build out the central policy server, or take the project in a new direction. You don't need permission.

## If You Want to Be a Maintainer

If you fork this and build something worth maintaining — and you want to be recognized as the upstream home for an actively developed version of HookMon — reach out. If a community forms around a fork and someone wants commit rights or to be named a project lead, that conversation is worth having. The goal is for this work to have a life beyond its initial release, not to gate that life on any single person's bandwidth.

## What This Repo Is Not

- An actively supported open source project
- A product with a vendor behind it
- A commitment to fix CVEs, respond to issues, or merge contributions on any timeline
